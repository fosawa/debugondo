OPNLIB関数一覧

●opncheck

[書式]
  int opncheck(void)
[機能]
  OPNDRVの存在確認をします。
[解説]
  OPNDRVが常駐しているかどうかを調べ、バージョン番号を100倍した値を返します。
[戻り値]
  OPNDRVが常駐していれば、バージョン番号の100倍の値を、常駐していなければ0を返します。

●getdivision

[書式]
  int getdivision(unsigned char far *dat)
[機能]
  OPN形式データのdivision値を得ます。
[解説]
　与えられたデータのdivision値(=4分音符の分解能)を得ます。もし、与えられたデータがOPN形式でない場合には、0を返します。
[戻り値]
　division値が返ります。OPN形式でない場合には0が返ります。

●getopnver

[書式]
  int getopnver(unsigned char far *dat)
[機能]
  OPN形式データのバージョンを得ます。
[解説]
　与えられたデータがMML2OPN.EXEのどんなバージョンで作られたかを調べます。
　データバージョンの100倍値が返ります。もし、与えられたデータがOPN形式でない場合には、0を返します。
[戻り値]
　データバージョンの100倍が返ります。OPN形式でない場合には0が返ります。

●opnstart

[書式]
  int opnstart(unsigned char far *dat)
[機能]
  OPN形式データの演奏を開始します。
[解説]
　付属のユーティリティーMML2OPN.EXEで作成されたOPN形式のデータを演奏します。datは、OPN形式データを差すポインタです。
　この関数を呼び出すと同時に、演奏が開始されます。演奏終了までOPN形式データの入っている配列を破壊してはなりません。
[戻り値]
　正常に演奏を受け付けた時には、OPNデータバージョンの100倍値を、OPNデータでなかった場合には、0を返します。

●opnstart2

[書式]
  int opnstart2(unsigned char far *dat, unsigned long length)
[機能]
  付属のユーティリティーMML2OPN.EXEで作成されたOPN形式のデータを常駐演奏させます。datは、OPN形式データを差すポインタで、lengthはそのdatの長さです。
  常駐演奏にあたっては、OPNDRVを常駐させる際に、/fオプションでいくらかのバッファを確保しておく必要があります。
[戻り値]
  与えられたデータがOPN形式でない場合には1、バッファが足りない場合には-1、正常終了には0が返ります。

●mend

[書式]
  void mend(void)
[機能]
  演奏を終了します。
[解説]
　関数opnstartやopnstart2で演奏開始させたものを終了させます。
[戻り値]
　ありません。

●mfade

[書式]
  void mfade(int fmspeed, int psgspeed)
[機能]
  OPN演奏をフェードアウトします。
[解説]
　演奏中の曲にフェードアウトをかけます。fmspeedは、FM音源およびリズム音源のフェードアウトスピード、psgspeedは、PSG音源のフェードアウトスピードです。値が大きいほど、ゆっくりとフェードアウトします。
　フェードアウト後も、演奏終了にはなりません。演奏終了には、mend関数を使用してください。
[戻り値]
　ありません。

●varread

[書式]
  int varread(int num)
[機能]
  OPNDRVの内部変数値を得ます。
[解説]
  内部変数の値を得ます。numで、内部レジスタ番号を指定します。0<=num<=99です。
[戻り値]
  指定された内部変数の値を返します。内部変数は1バイトですから、戻り値は0から255の間の数になります。

●tempochange

[書式]
  void tempochange(int tempo, int division)
[機能]
  テンポを変えます。
[解説]
　演奏中の曲のテンポをtempoに変更します。単位は、1分間当たりの4分音符の数です。変数divisionには、曲データのdivisionを指定してください(getdivision関数で取得可能です)。
　なお、曲データ中にテンポ設定命令が出てくると、そちらのテンポに変更されます。
[戻り値]
　ありません。

●regread

[書式]
  int regread(int regno, int sel)
[機能]
  FM音源チップ内のレジスタ値を得ます。
[解説]
　regnoで指定されたFM音源チップ内のレジスタ値を得ます。selで、YM2203とYM2608のレジスタ選択を行ないます。sel=0のとき、YM2203のレジスタが、sel=1のとき、YM2608のレジスタが選択されます。
　この関数は、FM音源チップ内からのデータを読むわけではないので、OPNDRV以外の制御でFM音源チップ内レジスタを変更した場合は、無効です。
[戻り値]
　FM音源チップ内のレジスタの値が返ります。

●sestore

[書式]
  int sestore(unsigned char far *sdata, unsigned long length)
[機能]
  効果音(SE)をOPNDRVの内部バッファに常駐させます。
[解説]
  sdataで指定されたSEデータをOPNDRV内部のバッファにコピーします。データ長は、lengthで指定します。この関数でバッファにコピーした後、sestart2関数で、バッファにコピーされたSEを鳴らすことが可能になります。
  SEのバッファは、OPNDRVの常駐時に/sオプションで指定します。
[戻り値]
  内部バッファへの格納がうまく行なわれた場合には、SEデータバージョンの100倍値を、SEデータでなかった場合には0を、内部バッファが足りない場合には、-1を返します。

●midistart

[書式]
  int midistart(void huge *addr)
[機能]
  MIDIの演奏を開始します。
[解説]
　addrで指定されたデータをMIDIで演奏します。addrは、標準MIDIファイルデータを差している必要があります。
[戻り値]
　正常終了で0、演奏不能データ(標準MIDIファイルデータでないか、サポートしていないフォーマット)ならば1、インターフェースが使用不可(midiinitを呼んでいないなど)ならば2が戻ります。

[戻り値]

●getmidi

[書式]
  int getmidi(void)
[機能]
  MIDIステータスを調べます。
[解説]
　MIDIステータスを見ます。ステータスは以下の通りです。

　下位8bit: 0=MIDI演奏中、1=待機中、2=演奏終了
  上位8bit: ペロシティー補正値

　ベロシティー補正値とは、現在の音量比率を指します。ボリューム比率は、ベロシティー補正値/128であり、ベロシティー補正値が128であれば通常ボリューム、256であれば、通常ボリュームの2倍ということになります。このステータスは、主にフェードアウトコマンド送出後、フェードアウトが完了したかどうかを調べるのに使います。詳しくは、midifade関数の説明をご覧ください。
[戻り値]
　上記のステータスが戻ります。

●midifade

[書式]
  void midifade(unsigned char start, unsigned char end, unsigned int speed)
[機能]
  フェードイン/アウト処理をします。
[解説]
　speed(単位はgatetime)(!=0)で指定した時間内で、音量をstart/128倍からend/128倍に直線的変化をさせます。
　start=128、end=0とすれば、フェードアウトし、逆にstart=0、end=128とすればフェードインします。フェードイン/アウト終了後もこのボリューム補正値は持続しますので、通常ボリュームでの演奏に戻したい場合には、midifade(128,128,1)とする必要があります。
[戻り値]
　ありません。

●pcmplay

[書式]
  int pcmplay(unsigned char huge *dat)
[機能]
  PCM再生します。
[解説]
　datで指定したデータをPCM音源で再生します。データは、添付のWAV2PCM.EXEなどによりコンバートされたPCM形式データです。
　関数を呼んでから、実際に発音されるまでには多少時間がかかりますから注意して下さい。
　PCM演奏終了まで、datの内容を破壊してはいけません。
[戻り値]
　PCMデータであれば、PCMデータのバージョンの100倍値が返ります。そうでない場合には、0が戻ります。

●pcmplay2

[書式]
  int pcmplay(unsigned char huge *dat, unsigned long length)
[機能]
  PCMを常駐再生させます。
[解説]
  datで指定したデータをPCM音源で再生します。その際、OPNDRVの内部バッファに一旦コピーされてから再生されます。バッファサイズは、OPNDRV常駐時に、/pオプションで指定します。
  データは、添付のWAV2PCM.EXEなどによりコンバートされたPCM形式データです。lengthでそのデータの長さを指定します。
　関数を呼んでから、実際に発音されるまでには多少時間がかかりますから注意して下さい。
[戻り値]
  PCMデータであれば、PCMデータのバージョンの100倍の値が返ります。バッファが足りない場合には、-1が返ります。PCMデータでなければ、-2が返ります。

●getstatus

[書式]
  int getstatus(void)
[機能]
  各種ステータスを得ます。
[解説]
　OPNDRVの各種ステータスを見ます。ステータスは以下の通りです。

  ビット0: FM音源再生 (0:再生中,1:停止中)
  ビット1: PCM音源再生(0:再生中,1:停止中)
  ビット4: ポート番号が(0:188h, 1:288h)
